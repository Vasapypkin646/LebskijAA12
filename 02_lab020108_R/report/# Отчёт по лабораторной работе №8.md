## Ход работы  

### 1. Теоретическая часть  

**Визуальное программирование** - это методология программирования, в которой программы создаются путем манипулирования графическими элементами, а не написания текстового кода. 

**Язык Scratch** - это визуальная среда программирования, разработанная для обучения основам программирования. Основные особенности:

- **Блочное программирование** - использование готовых блоков для создания программ
- **Спрайты** - графические объекты, которые могут выполнять различные действия
- **Сцены (фоны)** - различные уровни или состояния программы
- **Событийное программирование** - реакция на действия пользователя и другие события

**Основные концепции Scratch:**
- Спрайты и их костюмы
- Блоки управления (события, контроль, движение и т.д.)
- Система координат и направлений
- Переменные и списки
- Вещание (broadcast) для взаимодействия между спрайтами

### 2. Практическая часть  

#### Задание 1: Создание игры в Scratch

Создана игра с тремя активными спрайтами и системой уровней.

**Описание игры:**
Игра представляет собой платформер, где игрок управляет красным мячом, собирает яблоки и избегает движущегося куба. При сборе всех яблок происходит переход на следующий уровень.

**Реализованные спрайты:**

#### Спрайт 1: Красный мяч (управляемый персонаж)
- **Роль:** Главный персонаж, управляемый игроком
- **Управление:** Стрелки клавиатуры (вверх, вниз, влево, вправо)
- **Особенности:** Может прыгать, двигаться по платформам, собирать яблоки

**Скриншот кода красного мяча:**

![Код мяча](./код_мяча.png)


**Основные блоки управления:**
- При нажатии клавиш стрелок - движение в соответствующих направлениях
- Проверка столкновений с платформами
- Обработка сбора яблок
- Управление анимацией и внешним видом

#### Спрайт 2: Куб (автономный персонаж)
- **Роль:** Препятствие, движущееся независимо от игрока
- **Поведение:** Перемещение между заданными точками по траектории
- **Особенности:** Постоянное движение, создает сложность для игрока

**Скриншот кода куба:**

![Код куба](./код_куба.png)

**Основные блоки управления:**
- Бесконечный цикл движения между точками A и B
- Проверка столкновений с границами сцены
- Изменение направления при достижении крайних точек
- Постоянная скорость движения

#### Спрайт 3: Яблоко (интерактивный объект)
- **Роль:** Собираемый предмет, меняющий поведение при взаимодействии
- **Поведение:** Исчезает при касании с мячом, появляется в новых местах
- **Особенности:** Реагирует на действия игрока, влияет на прогресс уровня

**Скриншот кода яблока:**

![Код яблока](./код_яблока.png)

**Основные блоки управления:**
- Ожидание касания с мячом
- Исчезновение при сборе
- Подсчет собранных яблок

#### Система уровней
Реализована система из 2 уровней через смену фонов:

**Уровень 1: Начальный уровень**
- Простая платформа
- Медленно движущийся куб

**Уровень 2: Продвинутый уровень**
- Сложная структура платформ
- 1 яблоко для сбора

- При столкновении с краем уровня - переход на следующий
- Смена фона (заднего плана)
- Сброс позиций спрайтов
